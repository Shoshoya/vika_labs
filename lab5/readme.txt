Task 1: Зловмисний проксі-сервер
Було створено проксі-сервер на порту 8080, який перенаправляв запити на основний сервер (Port 3000). У режимі Breach Mode він перехоплював усі вхідні запити та виводив у термінал заголовок Cookie. У результаті, навіть попри наявність атрибуту HttpOnly, проксі успішно отримав значення SessionID. Це довело, що HttpOnly захищає cookie лише від доступу через JavaScript у браузері, але не від зловмисника, який контролює мережевий шлях, адже проксі бачить сирі HTTP-заголовки.
Task 2: Посилення захисту прапорцем Secure
У маршруті /login на основному сервері до заголовка Set-Cookie було додано атрибут Secure. Після цього при доступі через проксі (http://localhost:8080) браузер перестав надсилати cookie. Це підтвердило, що прапорець Secure гарантує передачу конфіденційних токенів лише через зашифровані з’єднання (HTTPS). Таким чином, він захищає дані від перехоплення у відкритому тексті. Оскільки наше з’єднання з проксі було по протоколу HTTP, браузер відмовився передавати cookie з міркувань безпеки.
